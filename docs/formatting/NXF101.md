# NXF101 — Specific Block Closers

**Category:** Closer style  

## Rule

Every block closer must use the **specific tag** matching its opener.
The generic closer `__` is accepted by the parser but normalized to the specific form
by the formatter. Pass `--permissive` to preserve the original closer style instead.

| Opener                            | Specific closer |
|-----------------------------------|-----------------|
| `frame`                           | `__frame`       |
| `action`                          | `__action`      |
| `deliberate`                      | `__deliberate`  |
| `plan`                            | `__plan`        |
| `body`                            | `__body`        |
| `in`                              | `__in`          |
| `out`                             | `__out`         |
| `guidelines`                      | `__guidelines`  |
| `repeat`                          | `__repeat`      |
| `if` / `elif` / `else`            | `__if`          |
| `toolset`                         | `__toolset`     |
| `do` (block form)                 | `__do`          |
| `from toolset … use` (block form) | `__use`         |

## Rationale

Specific closers make it immediately clear which block is being closed when reading
nested or lengthy blocks, without requiring the reader to count indentation levels.

## Examples

**Compliant:**

```nxs
action greet >> greets the user <<:
  body:
    >> say hello <<
  __body
__action
```

**Non-compliant (normalized by formatter unless `--permissive`):**

```nxs
action greet >> greets the user <<:
  body:
    >> say hello <<
  __
__
```

## Autofix

No text-level fix is provided. The formatter corrects closer style automatically during
the AST round-trip (`format` mode, `strict=True`). In `check` mode the violation is
reported but no inline fix is attached.
