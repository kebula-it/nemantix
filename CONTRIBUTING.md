# How to contribute to Nemantix

Thank you for considering to contributing to Nemantix. 
Before starting, make sure the read our [documentation](docs/00%20-%20Welcome%20to%20Nemantix.md) and 
familiarize with the [example scripts](https://github.com/kebula-it/nemantix/tree/main/examples).

## Bugs and Issues
If you found a bug, look for an open issue addressing the problem. 
If there are no issue about that, [open a new one](github.com/kebula-it/nemantix/issues/new).
Be sure to include:
* a **title and clear description** with as much relevant information as possible;
* a **code sample** or an **executable test case**: in terms of Nemantix scripts and the Python 
code that calls them, demonstrating the expected behavior that is not occurring.

## Pull Requests
If you wrote a patch that fixes a bug:
* Open a new GitHub pull request with the patch: the patch should include that appropriate test cases.
* Ensure the PR description clearly describes the problem and solution. Include the relevant issue number if applicable.
* Submit the PR for approval.

> **NOTE:** before opening a new PR, fork the repo and create a new branch from `main`.

## Feature request
If you want to propose a new Nemantix feature:
* Open an issue with appropriate title and description;
* Describe what the feature is about and why it is relevant; 
* Provide an example use case for the intended feature.

## Documentation
Contributions to the Nemantix documentation are welcome. 
In particular, you can:
* Write new example scripts: either educational or small applicative use cases.
