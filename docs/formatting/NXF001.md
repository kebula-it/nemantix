# NXF001 — Indentation

**Category:** Whitespace

## Rule

Each indentation level uses **2 spaces**. Tabs are not permitted.

## Rationale

NXS blocks nest multiple levels deep (e.g. `deliberate` → `plan` → `body` → `repeat` →
`if`). 4-space indentation produces overly wide lines at deep nesting; two spaces keep 
code readable within the 120-character line limit ([NXF002](NXF002.md)).

## Examples

**Compliant:**

```nxs
action greet >> greets the user <<:
  in:
    name >> the user's name <<
  __in
  body:
    >> say hello to [name] <<
  __body
__action
```

**Non-compliant (4-space indent):**

```nxs
action greet >> greets the user <<:
    in:
        name >> the user's name <<
    __in
    body:
        >> say hello to [name] <<
    __body
__action
```

## Autofix

When the indentation unit is not 2 spaces (e.g. 4-space), the formatter rewrites the
entire file: every indentation level is re-expressed as multiples of 2 spaces.
The fix is applied as a single whole-file edit and is always provided.
