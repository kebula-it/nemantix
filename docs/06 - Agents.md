# The Nemantix Agent

The **Agent** is the main entry point for executing Nemantix scripts in response to a user request. The agent can answer two types of requests:

* **Coded:** expressed as a `do-statement`, requiring the execution of a specific deliberate, action, tool, or built-in function.
* **Uncoded:** a natural language query that expresses the user's intent. In this case, the agent determines if the request can be answered. If so, it selects the best deliberate and extracts possible input arguments embedded within the request.

## Agent class
```python
def __init__(self, expertise: Expertise, llm_proxy: AbstractLLMProxy | None = None,
                 external_vars: dict[str, Any] | None = None, use_embedder=False,
                 use_knowledge_base=False, build_on_start=True,
                 kb_config: KnowledgeBaseConfig | None = None, log_level=None):
```
Instantiates an Agent:
- `expertise`: The expertise that the Agent uses to execute tasks.
- `llm_proxy`: The LLM proxy used for natural language processing (including a proxy for OpenAI or others). If None,
it uses the LLM proxy used by the provided Expertise.
- `external_vars`: external variables visible to scripts, used for holding secrets, configurations, etc.
External variables are accessible within an NXS script through a special `ENV` identifier: for example,
an external variable named `secret` can be accessed by `[ENV:secret]`. 
> The content of external variables associated with the special key `secrets` is only visible by tools of a `Toolset`.
> These "obfuscated" variables can be defined as follows: 
```python
external_variables=dict(db_url="db:url", 
                        secrets=dict(password='obfuscated_string'))

# do print using [[ENV:db_url]]  # prints db:url  
# do print using [[ENV:password]]  # prints Secret(name=password)  
```
- `use_embedder`: enables the use of embedding models to compute textual similarity. If not specified, the similarity 
is evaluated by the LLM proxy.
- `use_knowledge_base`: if True, enables the use of the knowledge base. 
- `build_on_start`: if True, the expertise is built automatically at startup coding the scripts if necessary.
- `kb_config`: configuration params for instantiating th Knowledge base. Mandatory if `use_knowledge_base=True`.
- `log_level`: the global logging level.

### Method `run`
```def run(self, user_request: str, schema=None, **kwargs) -> tuple[Exception, Any]```
Executes the part of the script relevant to the provided request:
- `user_request`: a string representing either a coded or uncoded request.
- `schema`: an optional Pydantic schema that represents how the output response should be structured.
- `kwargs`: additional execution arguments.

Returns a tuple containing:
- An error object, if present;
- The output values generated by the request.

## Agent State

The Agent has a state that can be accessed and modified within scripts via the `STATE` identifier. For example,
`[STATE:key]` accesses the value associated with the key field. Additionally, the instruction `[[STATE:other] = "value"]`
changes the agent's state by adding a new field other.
The state is reused across requests.

## Output Response
The output generated from script execution can be formatted to adhere to a given schema. The schema is represented by a Pydantic class definition, for example:
```python
from pydantic import BaseModel

class Ticket(BaseModel):
    label: str
    language: str
    summary: str

class OutputSchema(BaseModel):
    ticket: Ticket
    reply: str
```

## Example of Instantiating the Agent
The following example shows how to instantiate and use the Agent with an NXC file and example credentials.
```python
import json
import logging

from pathlib import Path
from nemantix.core import Agent, Expertise
from nemantix.security import Verifier

# Create the Expertise instance
exp = Expertise.from_local_scripts(paths=['/path/to/your/file.nxs'],
                                   verifier=Verifier('public-key-path'),
                                   credentials_path='credentials.json')

# Create the Agent
agent = Agent(expertise=exp, build_on_start=True)

# Execute the request
err, out = agent.run(user_request="Assemble a castle ladder")

print("Error:", err)
print("Output:", out)
```
### Coded request
```python
agent = Agent(expertise=exp, build_on_start=True)

# Execute the coded request
error, output = agent.run(user_request='do action do_example using ["3"] producing [[result]]')

# Handle the error and print the output
if error:
    print(f'Error occurred: {error}')
else:
    print(output)  # output will be 30
```

### Uncoded request
```python
agent = Agent(expertise=exp, build_on_start=True)

# Execute a natural language request with an output format
_, out = agent.run(user_request='Summarize the ticket "Ticket-BUG <fix necessary for infrastructure orchestration code>"',
                   schema=OutputSchema)

print(out)
# > OutputSchema(ticket=Ticket(label=...), reply="...")
```

### Deliberation fallback
For both coded and uncoded request a **fallback** mechanism can
be optionally enabled, allowing the coding of a new, not planned deliberate 
for the actual request (assuming no other deliberate can answer it):
```python
# the fallback mechanism is enabled by the Expertise
exp = Expertise.from_local_scripts(paths=['/examples/ticket.nxs'],
                                   verifier=Verifier('public-key-path'),
                                   allow_fallback_deliberate=True,
                                   credentials_path='credentials.json')
agent = Agent(expertise=exp, build_on_start=True)

# The request cannot be answered by the deliberates defined in examples/ticket.nxs
# therefore a new deliberate will be coded.
_ = agent.run(user_request='Send the ticket "Ticket-BUG <fix necessary for infrastructure orchestration code>"'
                            'to the IT department at IT@department.com')

```
