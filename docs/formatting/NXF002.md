# NXF002 — Line Length

**Category:** Line length  

## Rule

No line may exceed **120 characters**.

When a line exceeds this limit, the formatter switches to the multi-line form
of the construct (see [NXF201](NXF201.md), [NXF401](NXF401.md), [NXF402](NXF402.md)).

## Rationale

120 characters accommodate deep nesting with 2-space indentation ([NXF001](NXF001.md))
while remaining readable on standard widescreen displays without horizontal scrolling.

## Examples

**Compliant — prompt fits inline:**

```nxs
do llm using [[query]] >> summarise the content in a single concise sentence << producing [[result]]
```

**Compliant — `do` expanded to block form because the inline version exceeds 120 characters:**

```nxs
do llm:
  using [[query]]
  producing [[result]]
__do >> summarise the content of the document focusing on the key findings <<
```

**Compliant — standalone prompt word-wrapped to block form:**

```nxs
    >>>
summarise the content of the document focusing on the key findings and
omitting any introductory or concluding remarks
    <<<
```

## Notes

- The limit applies to the rendered line including leading indentation spaces.
- Blank lines and comment lines (`# ...`) are exempt.

## Autofix

For standalone `>> text <<` prompt lines, the formatter word-wraps the content to
`>>>\n...\n<<<` block form, breaking at the last word boundary before column 120.

Inline `do` statements and `from toolset` imports that exceed 120 characters are handled
by [NXF402](NXF402.md) and [NXF401](NXF401.md) respectively — NXF002 defers to those
more specific rules and does not report a separate violation for those lines.

For all other long lines (long identifiers, expressions, annotations) no autofix is provided.
