# NXF004 — Blank Lines Between Internal Sections

**Category:** Blank lines  

## Rule

Exactly **one blank line** must separate consecutive sections within a block.
Internal sections are: `in`, `out`, `body`, `guidelines`, and nested `frame` or `action` blocks.

## Rationale

Sections within a block represent distinct phases of an action's contract (inputs,
outputs, implementation). A single blank line provides visual separation between them
without interrupting the reading flow of the overall block.

## Examples

**Compliant:**

```nxs
action process >> processes input <<:
  in:
    data >> the input data <<
  __in

  out:
    result >> the processed result <<
  __out

  body:
    >> transform [data] into [result] <<
  __body
__action
```

**Non-compliant — no blank line between sections:**

```nxs
action process >> processes input <<:
  in:
    data >> the input data <<
  __in
  out:
    result >> the processed result <<
  __out
  body:
    >> transform [data] into [result] <<
  __body
__action
```

## Notes

- Blank lines are not added between the block header and its first section, nor between
  the last section and the block closer (see [NXF005](NXF005.md)).
- The rule applies recursively to nested blocks.

## Autofix

- **Missing blank line** between a section closer and the next section header — inserts one.
- **Extra blank lines** — removes the surplus, leaving exactly one.
