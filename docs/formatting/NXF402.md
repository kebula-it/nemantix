# NXF402 — `do` Statement Form

**Category:** Statement style  

## Rule

The `do` statement form must match the line length constraint ([NXF002](NXF002.md)):

- Use the **inline** form when the statement fits within 120 characters.
- Use the **block** form when the inline form would exceed 120 characters.

## Forms

**Inline:**

```nxs
do llm using [[query]] producing [[result]]
do tool search using [[q] = [query]] producing [[hits]]
```

**Block:**

```nxs
do llm:
  using [[query]]
  producing [[result]]
  >> use the context to answer the question precisely and concisely <<
__do
```

## Examples

**Compliant — inline (fits within 120 chars):**

```nxs
do llm using [[prompt]] producing [[answer]] >> answer the user <<
```

**Compliant — block (exceeds 120 chars inline):**

```nxs
do llm:
  using [[context] = [[document] | " — " | [question]]]
  producing [[answer]]
  >>> answer the question based solely on the provided context,
      citing the relevant passage when possible <<<
__do
```

**Non-compliant — block form used when inline would fit:**

```nxs
do llm:
  using [[query]]
  producing [[result]]
__do
```

## Autofix

- **Block → inline**: when the block form fits within 120 characters, collapses it to the inline form.
- **Inline → block**: when the inline form exceeds 120 characters, expands it to the block form.
