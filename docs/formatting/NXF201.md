# NXF201 — Prompt Style

**Category:** Prompt style  

## Rule

Use the **inline** form `>> text <<` when the entire line (including indentation and
surrounding statement) fits within 120 characters ([NXF002](NXF002.md)).

Use the **block** form `>>> text <<<` when the prompt would cause the line to exceed
120 characters, or when the prompt text itself contains newlines.

## Forms

| Form   | Syntax                       | When                                       |
|--------|------------------------------|--------------------------------------------|
| Inline | `>> single line text <<`     | line ≤ 120 chars and no newlines           |
| Block  | `>>> multi or long text <<<` | line > 120 chars or text contains newlines |

## Examples

**Compliant — inline:**

```nxs
action greet >> greets the user by name <<:
```

**Compliant — block (exceeds 120 chars inline):**

```nxs
action summarise
  >>> produces a concise summary of the provided document,
      focusing on key findings and omitting boilerplate <<<:
```

**Compliant — block (multiline content):**

```nxs
deliberate Assistant when
  >>> the user sends a message that requires
      processing across multiple steps <<<:
```

**Non-compliant — inline prompt with newline in text:**

```nxs
>> first line
   second line <<
```

## Autofix

When a block prompt `>>> text <<<` appears entirely on a single line and the equivalent
inline form `>> text <<` fits within 120 characters, the formatter collapses it to the
inline form.
