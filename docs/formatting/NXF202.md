# NXF202 — Microprompt Delimiter Spacing

**Category:** Prompt style

## Rule

The opening delimiter `>>` must always be followed by exactly one space, whether the
microprompt is closed (`>> content <<`) or open-ended (`>> content`).

The closing delimiter `<<` must always be preceded by exactly one space.

For multiline microprompts (`>>> ... <<<`), the spacing rule does not apply.

## Rationale

Inconsistent spacing (`>>content<<`, `>>content <<`, `>> content<<`) reduces
readability. The canonical form `>> content <<` is enforced automatically by the
formatter via AST round-trip; this rule makes the requirement explicit and
detectable in check mode.

## Examples

**Compliant (closed):**

```nxs
action classify >> classifies intent <<:
  body:
    >> determine category <<
  __body
__action
```

**Compliant (open-ended):**

```nxs
do llm using [[prompt]] producing [[result]] >> summarise the content
```

**Non-compliant (closed — missing spaces):**

```nxs
    >>determine category<<
```

**Non-compliant (open-ended — missing space after `>>`):**

```nxs
do llm using [[prompt]] producing [[result]] >>summarise the content
```

**Compliant (multiline — no spacing rule):**

```nxs
action classify >> classifies intent <<:
  body:
    >>> determine category
        based on context <<<
  __body
__action
```

## Autofix

Inserts the missing space after `>>` and/or before `<<` on the offending line.
