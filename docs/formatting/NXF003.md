# NXF003 — Blank Lines Between Top-Level Blocks

**Category:** Blank lines  

## Rule

Exactly **one blank line** must separate consecutive top-level blocks.
Top-level blocks are: `frame`, `action`, `deliberate`, `toolset`, and `require` statements.

## Rationale

Top-level blocks represent distinct semantic units. One blank line provides visual
separation without excessive whitespace; zero lines blend adjacent blocks together,
two or more add noise without conveying additional structure.

## Examples

**Compliant:**

```nxs
require ./utils.nxc

frame Person:
  slot name as TEXT
__frame

action greet >> greets the user <<:
  body:
    >> say hello <<
  __body
__action

deliberate Assistant when >> user sends a message <<:
  plan:
    body:
      do action greet
    __body
  __plan
__deliberate
```

**Non-compliant — no blank line between blocks:**

```nxs
frame Person:
  slot name as TEXT
__frame
action greet >> greets the user <<:
  body:
    >> say hello <<
  __body
__action
```

**Non-compliant — two blank lines between blocks:**

```nxs
frame Person:
  slot name as TEXT
__frame


action greet >> greets the user <<:
```
