# NXF501 — Annotation Indentation

**Category:** Annotation style  

## Rule

Annotations (`@key: value`) and labels (`{label}`) that form an intentable prefix
must be indented at the **same level as the block they introduce**.

## Rationale

Annotations are syntactically part of the block header. Aligning them with the block
keyword makes the association visually unambiguous, especially when multiple annotations
precede a deeply nested block.

## Examples

**Compliant:**

```nxs
@completion: drafted->frozen
@intent.goal: classify the user intent
action classify >> classifies user intent <<:
  body:
    >> determine the category of [input] <<
  __body
__action
```

**Compliant — nested block:**

```nxs
action outer >> outer action <<:
  body:
    @completion: drafted->frozen
    repeat >> until done <<:
      >> do work <<
    __repeat
  __body
__action
```

**Non-compliant — annotation over-indented:**

```nxs
action classify >> classifies user intent <<:
  body:
      @completion: drafted->frozen
    repeat >> until done <<:
      >> do work <<
    __repeat
  __body
__action
```

## Autofix

Re-indents the annotation line to match the indentation level of the block it introduces.
